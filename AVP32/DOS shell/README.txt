Основные особенности пакета:

	В ходе работы AVP проводит тестирование и удаление вирусов из:

- Оперативной памяти.

- Файлов (включая архивные и упакованные);

- Системных секторов, содержащих Master Boot Record и загрузочный сектор
  (Boot-сектор).

	К основным особенностям AVP можно отнести:

- Возможность детектирования и удаления огромного числа самых разнообразных
  вирусов, в том числе:

        - полиморфных или самошифрующихся вирусов;

        - стелс-вирусов или вирусов-невидимок;

        - "троянских коней";

        - новых вирусов, написанных для  Windows 3.XX и Windows 95;

        - макро-вирусов, заражающих документы Word и таблицы Excel,
          включая вирусы для Microsoft Office 97;

- Тестирование и обеззараживание оперативной памяти.

- Эвристический модуль Code Analyzer, позволяющий детектировать неизвестные
  вирусы. В соответствии с рядом тестов этот эвристический алгоритм
  позволяет обнаруживать более 80% новых вирусов.

- Возможность проверки локальных, сетевых, флоппи-дисков, CDROM дисководов.

- Возможность сканирования внутри упакованных файлов (модуль Unpacking
  Engine). Позволяет тестировать "на лету" файлы, упакованные наиболее
  популярными упаковщиками, такими, как PKLITE, LZEXE, DIET и т.д.

- Возможность сканирования внутри архивных файлов (модуль Extracting
  Engine). Позволяет тестировать "на лету" файлы, упакованные наиболее
  популярными архиваторами, такими, как ZIP, RAR, ARJ и LHA.

- Поиск в режиме избыточного сканирования, что означает полное сканирование
  содержимого исследуемых файлов вместо стандартной обработки только "точек
  входа" (т.е. тех мест,  где начинается обработка программ системой).

Так же AVP для DOS32 проверяет на вирусы локальные почтовые ящики
пользователей электронной почты:

Microsoft Mail (Outlook, Exchange - файлы .PST и .PAB, тип архива MS Mail)
Microsoft Internet Mail (файлы .MBX, тип архива MS Internet Mail).

Проверяются как открытые, так и зашифрованные базы (защищенные паролем).
Текущая версия проверяет только персональные файлы Microsoft Mail, проверка
серверных вариантов баз будет подключена в следующих версиях.

При проверке AVP проверяет каждую запись в базах электронной почты, выделяет
"приаттаченные" файлы и сканирует их на вирусы. При этом поддерживаются
форматы:

 UUEncode (both formats and all variants)
 XXEncode
 btoa (up to 5.0 - no Filename)
 btoa 5.*
 BinHex 4.0
 ship
 NETRUN 3.10
 NETSEND 1.0 (not packed)
 NETSEND 1.0C (packed)
 MIME base64

Также проверяются базы данных электронной почты форматов:

 Eudora Pro & Lite
 Pegasus Mail
 Netscape Navigator Mail
 JSMail SMTP/POP3 server (базу по пользователям)

При этом AVP также проверяются на присутствие "приаттаченных" данных (UUE,
XXE, и т.д.) все файлы, а не только базы данных электронной почты (т.е. AVP
не обращает внимания на формат файла).

Внимание! При включенной проверке почты скорость работы AVP может
значительно уменьшиться.

Комплект AVPLite32
------------------
AVPLite32 включает в себя следующие файлы:

README.TXT    - файл c описанием;
DOS4GW.EXE    - расширитель ДОСа
AVP32DOS.EXE  - программа AVPLite32
AVPDOS32.KEY  - кей-файл.
DEFDOS32.PRF  - файл настроек
AVP.INI       - файл конфигурации

Для работы AVPLite32 необходим набор антивирусных баз, который 
может поставляться отдельно. Антивирусные базы можно найти на сервере 
http://www.avp.ru 

Запуск AVPLite32 из командной строки и коды возврата (errorlevels)
----------------------------------------------------------------

Формат:  AVP32DOS [опции] [путь[\имя][...]]
	 Путь - любой путь DOS, * или *: - все жесткие диски
	 Имя - допустимы символы * и ?  По умолчанию - исполняемые файлы
Опции:

/-  удалять вирусы
/E  стирать зараженные файлы   /*  проверять все файлы
/M  пропустить тест ОЗУ        /N  проверять удаленные диски, используется с *:
/P  пропустить тест MBR        /1  проверять только одну дискету
/B  пропустить тест boot       /R  не проверять подкаталоги
/T=путь  путь для врем. файлов /U  запретить распаковку
/W[A][=имя файла]  отчет       /A  запретить разархивирование
  /WA добавлять                /H  запретить эвристический анализ
/O  выводить OK после имени    /@[!]=имя файла  проверять файлы по списку
/Y  пропустить диалоги           /@! удалять файл списка после проверки
/S  выключить звук             /D  один раз в день
/X  не использовать XMS        /V  избыточное сканирование
/MB проверять почтовые базы    /Z  запретить остановку во время тестирования
/MP проверять почтовые форматы /?  подсказка

Результаты сканирования доступны через коды возврата (DOS Errorlevel) при
использовании пакетных файлов:

  0 - вирусов не обнаружено;
  1 - сканирование не закончено;
  3 - найдены объекты, подозрительные на вирус;
  4 - обнаружен вирус;
  5 - все обнауженные виусы удалены;
  7 - файл AVP32DOS.EXE поврежден;
